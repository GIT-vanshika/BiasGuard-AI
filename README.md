**BiasGuard AI** is a comprehensive, end-to-end system designed to detect, explain, and monitor algorithmic bias in machine learning datasets and predictions. 

The project is split into two main components:
1. **Android Frontend App:** A modern mobile application built with Jetpack Compose that allows users to seamlessly upload datasets, view bias metrics, and read generated explanations on the go.
2. **FastAPI Backend:** A robust, cloud-hosted API that performs heavy statistical fairness computations and integrates with Google's Large Language Models (Gemini) for human-readable insights.

---

# 📱 Frontend: Android App

The mobile application acts as the user-facing dashboard for BiasGuard AI.

### Tech Stack
- **Language:** Kotlin
- **UI Framework:** Jetpack Compose (Modern Declarative UI)
- **Architecture:** MVVM (Model-View-ViewModel) + Repository Pattern
- **Networking:** Retrofit2 & OkHttp (Configured with 60s timeouts to handle backend cold starts)
- **Serialization:** Gson

### Features
- **File Uploads:** Securely upload CSV datasets directly from the Android device using `MultipartBody`.
- **Interactive Dashboards:** Visualizes complex bias metrics (Demographic Parity, Equal Opportunity).
- **LLM Insights Screen:** Displays executive summaries and counterfactual insights generated by Gemini.
- **Graceful Error Handling:** Dedicated `UiState` management (Idle, Loading, Success, Error) ensures smooth user experiences, even during cloud server wake-ups.

*(Note: The frontend code resides in its own Android Studio project directory.)*

---

# ⚙️ Backend: FastAPI Server

The backend acts as the computational engine for statistical analysis and AI integration.

### Tech Stack
- **Framework:** FastAPI (Python 3.11)
- **Data Processing:** Pandas, NumPy
- **Machine Learning:** Scikit-Learn
- **LLM Integration:** Google GenAI SDK (`google-genai`)
- **Hosting:** Render (via Docker)

### Core Capabilities
1. **Robust Data Parsing:** Safely processes CSV uploads, automatically mapping columns and handling dirty data.
2. **Fairness Metrics:** Calculates Demographic Parity and Equal Opportunity across protected attributes.
3. **Proxy Detection:** Identifies hidden proxy variables (e.g., zip code acting as a proxy for race).
4. **LLM Explanations:** Translates raw metric scores into actionable, human-readable insights using Google Gemini.
5. **Continuous Monitoring:** Tracks historical drift in bias scores.

### API Reference
| Method | Endpoint | Description | Payload Format |
|--------|----------|-------------|----------------|
| `POST` | `/upload-data` | Uploads a CSV dataset. | `multipart/form-data` |
| `POST` | `/run-bias-audit` | Calculates fairness metrics. | `application/x-www-form-urlencoded` |
| `POST` | `/get-explanations` | Generates LLM summaries. | `application/json` |
| `GET` | `/monitor-bias` | Retrieves historical drift data. | *None* |

## 🚀 Local Backend Setup

### 1. Clone & Install
```bash
git clone https://github.com/GIT-vanshika/BiasGuard-AI.git
cd BiasGuard-AI
python -m venv .venv
# Activate venv: `.venv\Scripts\activate` (Windows) or `source .venv/bin/activate` (Mac/Linux)
pip install -r requirements.txt
```

### 2. Environment Variables
Create a `.env` file from the template:
```bash
cp .env.example .env
```
Add your **Google Gemini API Key** to `.env`:
```env
GEMINI_API_KEY=your_actual_api_key_here
```

### 3. Run the Server
```bash
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
Interactive API docs available at `http://127.0.0.1:8000/docs`.

## ☁️ Cloud Deployment (Render)

This backend is actively deployed on Render using the included `Dockerfile`. 
**Important Note for Client Apps:** Because the backend is hosted on Render's free tier, the server goes to sleep after 15 minutes of inactivity. The first request after a sleep period (a **Cold Start**) can take up to 50 seconds. The Android frontend is specially configured to wait for this without crashing.
